7  Data Cleaning Pipeline

Author

Jacopo Olivieri

Published

July 17, 2025

7.1 Overview

The current data cleaning pipeline consists of 20 R scripts that process sewage discharge data, house price data, and spatial relationships to create datasets for analysis.

The pipeline is structured across 6 layers:

  1. Data Ingestion Layer (01_data_ingestion/): This layer contains scripts that retrieve or extract unprocessed data from external sources—such as downloading from APIs or unpacking zip archives. The scripts here do not clean or transform data; they only collect and save it locally.
  2. Data Cleaning Layer (02_data_cleaning/): This layer includes scripts that clean, reformat, and validate raw data files. These scripts ensure consistency across files by fixing column names, data types, missing values, and file formats. They produce uniform, structured outputs usable downstream.
  3. Data Enrichment Layer (03_data_enrichment/): This layer holds scripts that enhance cleaned data with additional structure—by generating lookup tables, calculating summaries, or tagging records with geographic or temporal metadata. These scripts prepare the data for matching and analysis by adding context.
  4. Feature Engineering Layer (04_feature_engineering/): This layer includes scripts that generate analysis-ready variables. This involves creating spatial matches, calculating distances, defining treatment indicators, and building other derived features required for regressions or causal analysis.
  5. Integration Layer (05_data_integration/): This layer contains scripts that merge datasets from different sources or time periods. These scripts align records across formats, join data on shared identifiers, and harmonise inconsistencies to produce a single dataset structure.
  6. Analysis Datasets Layer (06_analysis_datasets/): This layer includes scripts that build the final analysis datasets. These scripts define the unit of observation (e.g. house-level, site-level), assign exposure variables, and output clean panels or cross-sections used directly in empirical estimation.

7.3 Script Overview

7.3.1 1. Raw Data Ingestion Layer

EDM Data Standardisation (2021-2023)

  • script_name: edm_individ_data_standardisation_2021-2023.R
  • purpose: Standardises raw Event Duration Monitoring (EDM) data files from UK water companies for 2021-2023 by extracting, renaming, and organising files from zip archives.
  • inputs: ["data/raw/edm_data/EDM*.zip"]
  • outputs: ["data/raw/edm_data/YYYY_company_name_edm.*"]
  • suggested_layers: ["Raw Data Ingestion Layer"]

API Data Fetching (2024+)

  • script_name: fetch_edm_api_data_2024_onwards.R
  • purpose: Downloads and processes EDM data from ArcGIS API endpoints for 2024 onwards using pagination and saves as compressed JSON files.
  • inputs: ["scripts/config/api_config.R"]
  • outputs: ["data/raw/edm_data/raw_api_responses/company_name/*.json.gz", "output/log/edm_api_download_log.csv"]
  • suggested_layers: ["Raw Data Ingestion Layer"]

7.3.2 2. Data Cleaning & Standardisation Layer

Raw Data Conversion

  • script_name: convert_individ_raw_data_to_rdata_2021-2023.R
  • purpose: Converts and cleans individual raw EDM data files from CSV/Excel formats to standardised Parquet format for years 2021-2023.
  • inputs: ["data/raw/edm_data/YYYY_company_name_edm.*"]
  • outputs: ["data/processed/edm_data_2021_2023/*.parquet", "data/processed/edm_data_2021_2023/combined_edm_data_*.parquet"]
  • suggested_layers: ["Data Cleaning & Standardisation Layer"]

Individual EDM Data Combination

  • script_name: combine_individ_edm_data_2021-2023.R
  • purpose: Combines and cleans individual water company Parquet files from 2021-2023 into a single consolidated dataset with standardised datetime parsing.
  • inputs: ["data/processed/edm_data_2021_2023/*.parquet"]
  • outputs: ["data/processed/combined_edm_data.parquet"]
  • suggested_layers: ["Data Cleaning & Standardisation Layer"]

API Data Processing

  • script_name: process_edm_api_json_to_parquet_2024_onwards.R
  • purpose: Processes raw API JSON data into standardised Parquet files, extracts features data, and handles incremental updates for 2024 onwards.
  • inputs: ["data/raw/edm_data/raw_api_responses/company_name/*.json.gz"]
  • outputs: ["data/processed/edm_api_data/*.parquet"]
  • suggested_layers: ["Data Cleaning & Standardisation Layer"]

API Data Combination

  • script_name: combine_api_edm_data_2024_onwards.R
  • purpose: Combines and cleans individual water company API Parquet files from 2024 onwards into a single consolidated dataset.
  • inputs: ["data/processed/edm_api_data/*.parquet"]
  • outputs: ["data/processed/edm_api_data/combined_api_data.parquet"]
  • suggested_layers: ["Data Cleaning & Standardisation Layer"]

Annual Return Data Processing

  • script_name: combine_annual_return_data.R
  • purpose: Cleans and combines annual sewage spill return data from UK water companies for 2021-2024, standardising column names and formats.
  • inputs: ["data/raw/edm_data/YYYY_annual_return_edm.xlsx"]
  • outputs: ["data/processed/annual_return_edm.parquet"]
  • suggested_layers: ["Data Cleaning & Standardisation Layer"]

Consented Discharges Database Cleaning

  • script_name: clean_consented_discharges_database.R
  • purpose: Cleans the Environment Agency’s consented discharges database, standardising water company names and column mappings.
  • inputs: ["data/raw/ea_consents/consents_all.csv"]
  • outputs: ["data/processed/consent_discharges_db.RData", "data/processed/consent_discharges_db.csv"]
  • suggested_layers: ["Data Cleaning & Standardisation Layer"]

House Price Data Cleaning

  • script_name: clean_lr_house_price_data.R
  • purpose: Cleans and combines HM Land Registry UK property sales data for 2021-2023, enriching with postcode coordinates using PostcodesioR API.
  • inputs: ["data/raw/lr_house_price/pp-YYYY.csv"]
  • outputs: ["data/processed/house_price.parquet"]
  • suggested_layers: ["Data Cleaning & Standardisation Layer", "Data Enrichment & Aggregation Layer"]

7.3.3 3. Data Enrichment & Aggregation Layer

Annual Return Lookup Creation

  • script_name: create_annual_return_lookup.R
  • purpose: Builds a cross-year lookup table linking the same spill sites across annual EDM returns (2021-2024) using deterministic and probabilistic matching.
  • inputs: ["data/processed/annual_return_edm.parquet"]
  • outputs: ["data/processed/annual_return_lookup.parquet", "data/processed/annual_return_lookup_edges.parquet", "data/processed/annual_return_lookup.xlsx"]
  • suggested_layers: ["Data Enrichment & Aggregation Layer"]

Spill Statistics Aggregation

  • script_name: aggregate_spill_stats.R
  • purpose: Aggregates individual sewage overflow spills into monthly, quarterly, and yearly statistics using the 12/24 hour counting method.
  • inputs: ["data/processed/matched_events_annual_data/matched_events_annual_data.parquet"]
  • outputs: ["data/processed/spill_aggregated/agg_spill_yr.parquet", "data/processed/spill_aggregated/agg_spill_mo.parquet", "data/processed/spill_aggregated/agg_spill_qtr.parquet"]
  • suggested_layers: ["Data Enrichment & Aggregation Layer"]

Unique Spill Sites Creation

  • script_name: create_unique_spill_sites.R
  • purpose: Creates unique spill sites dataset by cleaning location data and parsing British National Grid coordinates from merged EDM data.
  • inputs: ["data/processed/matched_events_annual_data/matched_events_annual_data.parquet"]
  • outputs: ["data/processed/unique_spill_sites.parquet", "data/processed/unique_spill_sites.xlsx"]
  • suggested_layers: ["Data Enrichment & Aggregation Layer"]

7.3.4 4. Feature Engineering & Spatial Processing Layer

Spatial Matching

  • script_name: 10km_site_house_sale_match.R
  • purpose: Performs spatial matching between house sales and spill sites within 10km radius using spatial joins and distance calculations.
  • inputs: ["data/processed/house_price.parquet", "data/processed/unique_spill_sites.parquet"]
  • outputs: ["data/processed/spill_house_lookup.parquet"]
  • suggested_layers: ["Feature Engineering & Spatial Processing Layer"]

Spill Statistics Computing

  • script_name: compute_spill_stats.R
  • purpose: Calculates site-level spill statistics including cross-site thresholds and treatment indicators for monthly and quarterly periods.
  • inputs: ["data/processed/spill_aggregated/agg_spill_mo.parquet", "data/processed/spill_aggregated/agg_spill_qtr.parquet"]
  • outputs: ["data/processed/agg_spill_stats/"]
  • suggested_layers: ["Feature Engineering & Spatial Processing Layer"]

7.3.5 5. Integration Layer

Historical and API Data Integration

  • script_name: combine_2021-2023_and_api_edm_data.R
  • purpose: Harmonises and combines the processed 2021-2023 EDM data with API-sourced 2024+ data into a single consolidated dataset.
  • inputs: ["data/processed/combined_edm_data.parquet", "data/processed/edm_api_data/combined_api_data.parquet"]
  • outputs: ["data/processed/combined_edm_data.parquet"]
  • suggested_layers: ["Integration Layer"]

Individual and Annual Data Merging

  • script_name: merge_individ_annual_location.R
  • purpose: Merges location data from annual sewage overflow datasets with individual spill data using windfall, many-to-many, and fuzzy matching methods.
  • inputs: ["data/processed/combined_edm_data.parquet", "data/processed/annual_return_edm.parquet", "data/processed/annual_return_lookup.parquet"]
  • outputs: ["data/processed/matched_events_annual_data/matched_events_annual_data.parquet", "data/processed/matched_events_annual_data/events_unmatched.parquet", "data/processed/matched_events_annual_data/annual_unmatched.parquet", "data/processed/matched_events_annual_data/site_metadata.parquet"]
  • suggested_layers: ["Integration Layer"]

7.3.6 6. Master Dataset Assembly Layer

Cross-Sectional Database Creation

  • script_name: cross_section_db.R
  • purpose: Creates cross-sectional databases aggregating spill data: one for all years and one for 12 months prior to house sales, partitioned by radius thresholds.
  • inputs: ["data/processed/house_price.parquet", "data/processed/spill_house_lookup.parquet", "data/processed/spill_aggregated/agg_spill_mo.parquet"]
  • outputs: ["data/processed/agg_all_yrs_cross_section/", "data/processed/agg_12mo_cross_section/"]
  • suggested_layers: ["Master Dataset Assembly Layer"]

Site-Level Panel Data

  • script_name: site_panel.R
  • purpose: Creates site-level panel data by aggregating house price statistics (median, mean, weighted) for each spill site across multiple distance radii and time periods.
  • inputs: ["data/processed/house_price.parquet", "data/processed/spill_house_lookup.parquet"]
  • outputs: ["data/processed/dat_panel_site_level/"]
  • suggested_layers: ["Master Dataset Assembly Layer"]

House Panel Within Radius

  • script_name: house_panel_within_radius.R
  • purpose: Creates house sale-level panel data containing only houses within specified radius thresholds of spill sites, with monthly and quarterly aggregation.
  • inputs: ["data/processed/house_price.parquet", "data/processed/spill_house_lookup.parquet"]
  • outputs: ["data/processed/houses_within_radius_panel/"]
  • suggested_layers: ["Master Dataset Assembly Layer"]

General House Panel

  • script_name: house_panel_exp.R
  • purpose: Creates comprehensive house sale-level panel data including all houses (within and outside radius thresholds) with spill site relationship metadata for experimental analysis.
  • inputs: ["data/processed/house_price.parquet", "data/processed/spill_house_lookup.parquet", "data/processed/agg_spill_stats/"]
  • outputs: ["data/processed/general_house_panel_qtr/"]
  • suggested_layers: ["Master Dataset Assembly Layer"]
